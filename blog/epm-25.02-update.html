Survey Note: Detailed Analysis of Planning and ePBCS Updates for February 2025

This note provides a comprehensive analysis of the February 2025 updates (25.02) for Oracle’s Planning and Enterprise Planning and Budgeting Cloud Service (ePBCS). Written as of February 3, 2025, it leverages official documentation to offer actionable insights for businesses and consultants.

Background and Context

Oracle EPM Cloud continues to lead in enterprise performance management. Planning supports strategic decision-making, while ePBCS delivers robust financial planning. The 25.02 update builds on 25.01 with security enhancements, REST API improvements, and deeper Smart View integration.

Planning Updates

Dynamic Dashboard Filters with REST API
Adds dynamic filters to Planning dashboards with REST API support, allowing users to adjust data views via dropdowns (e.g., time, region) and automate filter application through API calls.
Impact: Enhances dashboard interactivity and automation capabilities.
Implementation Consideration: Update dashboards with filter configs and integrate REST API into automation scripts.
Business Benefit: Speeds up analysis with tailored, programmable data views.
REST API: Job Scheduling Enhancements
Expands the Planning REST API with new endpoints for scheduling and managing jobs (e.g., data loads, rule executions), offering finer control over timing and dependencies.
Impact: Improves automation flexibility for recurring tasks.
Implementation Consideration: Update job scripts to use new endpoints and test scheduling logic.
Business Benefit: Reduces manual oversight and optimizes workflow timing.

ePBCS Updates

Secure HTTPS Protocol Enforcement
Starting 25.02, all Connections screens and Update Connections REST APIs in ePBCS mandate HTTPS, phasing out HTTP URLs to enhance security, with a firm deadline set for March 2025.
Impact: Ensures encrypted data transfers across all connections.
Implementation Consideration: Update all HTTP URLs to HTTPS in Connections and APIs by March 2025.
Business Benefit: Strengthens data security and aligns with compliance standards.
Smart View: Standard Mode Default
Smart View’s Native Ad Hoc Behavior mode is deprecated, with Standard mode now default, offering enhanced features like zoom, pivot, and improved error handling for ePBCS grids.
Impact: Shifts users to a more robust ad hoc analysis framework.
Implementation Consideration: Switch to Standard mode in Application Settings and test worksheets for compatibility.
Business Benefit: Improves analysis capabilities and ensures future support.

Comparative Analysis and Business Implications

Product Update Impact Business Benefit
Planning Dynamic Dashboard Filters Enhanced interactivity Faster data analysis
Planning REST API: Job Scheduling Improved automation Optimized workflows
ePBCS HTTPS Enforcement Data encryption Improved security
ePBCS Smart View Standard Mode Robust analysis Future-proof workflows

Unexpected Detail: Filter Presets

Dynamic dashboard filters include preset options for common dimensions (e.g., last 12 months), an unadvertised feature that simplifies setup.

Important Actions and Considerations

Update all connection URLs to HTTPS by March 2025 to avoid disruptions.
Test Smart View worksheets in Standard mode to ensure compatibility.
Integrate new REST API job scheduling into automation scripts and validate dependencies.

Conclusion and Recommendations

The 25.02 updates strengthen security, automation, and analysis in Oracle EPM Cloud. Businesses should prioritize HTTPS transitions, Smart View testing, and REST API adoption. Contact WeRun Consulting for assistance.
