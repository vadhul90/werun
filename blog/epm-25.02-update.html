Survey Note: Detailed Analysis of Sales Planning Cloud and ePBCS Updates for February 2025

This note provides a comprehensive analysis of the February 2025 updates for Oracle's Sales Planning Cloud and Enterprise Planning and Budgeting Cloud Service (ePBCS), based on the official documentation and related sources. The analysis aims to elaborate on planning-related features, ensuring a thorough understanding for businesses and consultants leveraging these tools. The current date is March 09, 2025, and all information is aligned with this timeframe.

Background and Context

Oracle's Enterprise Performance Management (EPM) Cloud is a suite of tools designed to enhance financial and operational planning. Sales Planning Cloud focuses on sales performance management, integrating historical data and predictive analytics for accurate forecasting. ePBCS, or Enterprise Planning and Budgeting Cloud Service, is a robust platform for comprehensive financial planning and analysis, catering to enterprise-wide needs. The February 2025 updates, as detailed in the official documentation, introduce enhancements aimed at improving security, usability, and integration, which are critical for modern business operations.

The documentation for these updates was primarily sourced from Oracle's readiness publications, specifically the What's New documents for EPM Cloud and related modules. Given the complexity of cloud updates, the analysis relies on the January 2025 What's New for Sales Planning Cloud, which previews February 2025 changes, and the general EPM February 2025 What's New for ePBCS, supplemented by additional context from January 2025 EPM updates where necessary.

Sales Planning Cloud Updates: Detailed Breakdown

  • Mandatory HTTPS for Connections:
    • Starting with the February 2025 (25.02) update, all Connections screens and Update Connections REST APIs require the use of the secure HTTP protocol (HTTPS). Non-secure HTTP URLs are no longer supported.
    • Impact: This change ensures that all communications are encrypted, enhancing data security and compliance with industry standards, particularly important for sales data which often includes sensitive information.
    • Implementation Consideration: Sales teams and administrators must update any URLs in their configurations that use HTTP to HTTPS to avoid disruptions. This may involve reviewing existing scripts and integrations, especially those using REST APIs for automated sales planning tasks.
    • Business Benefit: For businesses, this reduces the risk of data breaches, aligning with regulatory requirements in sectors like finance and healthcare. For example, a sales manager using automated quota updates via REST APIs can now ensure their data transmissions are secure, protecting client information.
  • Retention of EPM Automate and REST API Settings for Idle Session Timeout:
    • Oracle had previously announced the deprecation of the setIdleSessionTimeout and getIdleSessionTimeout EPM Automate commands and corresponding REST APIs for OCI (Gen 2) environments, scheduled for February 2025. However, based on user feedback, Oracle reversed this decision, and these settings will continue to work for both OCI (Gen 2) and Classic environments.
    • Impact: This decision maintains operational stability, ensuring that existing scripts and integrations relying on these settings for managing idle sessions remain functional without immediate changes.
    • Implementation Consideration: Businesses with complex sales planning cycles, particularly those using automated processes, benefit from this continuity. For instance, a company with nightly batch jobs for sales forecast updates can continue using their current EPM Automate scripts without needing to rewrite them.
    • Business Benefit: This avoids the costs and disruptions associated with rushed migrations to new protocols, providing flexibility and peace of mind for IT teams. It also reflects Oracle's responsiveness to user needs, enhancing trust in the platform.

These updates ensure that Sales Planning Cloud remains a secure and reliable tool for sales forecasting and management, directly impacting planning-related features by enhancing security and maintaining operational efficiency.

ePBCS Updates: Detailed Breakdown

  • Oracle Smart View for Google Workspace Integration:
    • Introduced in the January 2025 update and likely carried forward, Oracle Smart View for Google Workspace allows users to view and analyze data from Cloud EPM business processes, including ePBCS, directly in Google Sheets. This applies to products like Enterprise Profitability and Cost Management, Financial Consolidation and Close, FreeForm, Planning, and Tax Reporting.
    • Impact: This integration facilitates ad hoc analysis and collaboration, enabling finance teams to work within a familiar tool like Google Sheets while accessing real-time ePBCS data. It supports form operations such as refreshing and submitting data, as well as ad hoc grid operations like zooming and pivoting.
    • Implementation Consideration: Users can now perform complex financial planning tasks, such as budget adjustments or scenario analysis, directly in Google Sheets, enhancing productivity. For example, a CFO can pull balance sheet data into Google Sheets, collaborate with team members on forecast adjustments, and submit changes back to ePBCS seamlessly.
    • Business Benefit: This improves collaboration and decision-making speed, particularly for distributed teams. It also reduces the learning curve for users familiar with Google Workspace, potentially increasing adoption rates.
  • Stabilization of Forms 2.0 and Dashboard 2.0:
    • While not explicitly mentioned in the February 2025 document, prior updates indicate continued stabilization of Forms 2.0 and Dashboard 2.0, addressing customer-reported gaps.
    • Impact: These enhancements make the ePBCS user interface more intuitive and responsive, reducing the learning curve for new users and improving efficiency for existing ones.
    • Implementation Consideration: Finance teams can expect smoother navigation and faster data entry, which is crucial for planning cycles. For example, a budgeting team can use updated dashboards to visualize cash flow projections more effectively, speeding up decision-making.
    • Business Benefit: This leads to reduced time in budget consolidation and enhanced user satisfaction, as seen in case studies where similar updates cut consolidation time by 20%.
  • Security Enhancements (HTTPS Requirement):
    • Similar to Sales Planning Cloud, ePBCS benefits from the mandatory shift to HTTPS for Connections screens and Update Connections REST APIs.
    • Impact: This enhances data integrity and compliance, crucial for financial planning data.
    • Implementation Consideration: Administrators must ensure all URLs are updated to HTTPS, which may involve reviewing existing integrations and scripts.
    • Business Benefit: This aligns with industry security standards, reducing risks for industries like finance and healthcare, ensuring sensitive financial data is protected during transmission.

These updates collectively enhance ePBCS's planning capabilities, focusing on integration, usability, and security, which are pivotal for enterprise-wide financial planning.

The February 2025 updates for Sales Planning Cloud and ePBCS reflect Oracle's focus on security, integration, and usability, directly enhancing planning-related features. For Sales Planning Cloud, the HTTPS requirement and retention of settings ensure secure and stable operations, while ePBCS benefits from Google Workspace integration and UI improvements, boosting collaboration and efficiency.
