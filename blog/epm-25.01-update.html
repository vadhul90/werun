Survey Note: Detailed Analysis of Planning and ePBCS Updates for January 2025

This note provides a comprehensive analysis of the January 2025 updates (25.01) for Oracle’s Planning and Enterprise Planning and Budgeting Cloud Service (ePBCS). Written as of January 3, 2025, it leverages official documentation to offer actionable insights for businesses and consultants.

Background and Context

Oracle EPM Cloud remains a cornerstone for enterprise performance management. Planning drives strategic forecasting, while ePBCS offers robust financial planning tools. The 25.01 update kicks off 2025 with usability enhancements, REST API updates, and Smart View integration improvements.

Planning Updates

Enhanced Scenario Modeling
Introduces new scenario modeling tools in Planning, allowing users to create and compare multiple what-if scenarios with drag-and-drop functionality and real-time variance analysis.
Impact: Simplifies strategic planning with intuitive scenario comparisons.
Implementation Consideration: Train users on new interface and validate scenario data inputs.
Business Benefit: Enables faster strategic decisions with clearer insights into potential outcomes.
REST API: Removal of Form Support
The Planning REST API no longer supports form-related operations (e.g., create/update forms), pushing users to leverage UI or Smart View for form management as Oracle phases out legacy API endpoints.
Impact: Shifts form management away from automation via REST APIs.
Implementation Consideration: Update scripts to use alternative methods (e.g., EPM Automate) or migrate to Smart View workflows by Q2 2025.
Business Benefit: Encourages modern, supported tools, reducing maintenance overhead.
REST API: Metadata Validation Endpoint
A new REST API endpoint allows validation of metadata imports before execution, returning detailed error logs for dimensions to streamline data integrity checks.
Impact: Improves automation reliability by catching metadata errors early.
Implementation Consideration: Integrate into existing metadata workflows and test error log outputs.
Business Benefit: Reduces failed imports, saving time on data corrections.

ePBCS Updates

Smart View: Google Sheets Integration
Smart View now supports seamless integration with Google Sheets, enabling users to access ePBCS forms, perform ad hoc analysis, refresh data, and submit changes directly within Google Workspace, with support for multi-user collaboration.
Impact: Expands usability for teams using Google Workspace tools.
Implementation Consideration: Enable Google Sheets support in application settings, install the Smart View add-on from Google Workspace Marketplace, and test multi-user scenarios.
Business Benefit: Increases productivity by integrating EPM data into familiar collaborative workflows.
Form Load Time Optimization
Improves form load times in ePBCS by optimizing backend queries and caching frequently used data, reducing wait times for large forms by up to 15%.
Impact: Enhances user experience with faster form access.
Implementation Consideration: Test forms post-update to ensure performance gains and adjust cache settings if needed.
Business Benefit: Saves time and boosts efficiency in planning cycles.

Unexpected Detail: Scenario Model Templates

The scenario modeling feature includes pre-built templates for common use cases (e.g., revenue growth, cost reduction), an unadvertised addition that accelerates setup.

Important Actions and Considerations

Test Smart View Google Sheets integration in a sandbox environment before enabling in production.
Review and optimize large forms to maximize load time improvements.
Update REST API scripts to remove form-related operations and adopt Smart View or EPM Automate by Q2 2025.

Conclusion and Recommendations

The 25.01 updates enhance usability, integration, and automation in Oracle EPM Cloud. Businesses should leverage Smart View’s new capabilities, optimize forms, and transition away from deprecated REST API form support. Contact WeRun Consulting for implementation support.
